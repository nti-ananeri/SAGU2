Este arquivo está dividido em duas(três?) partes: PENDENTES(, EM EXECUÇÃO?) e RESOLVIDOS. Sempre que o status de uma tarefa for modificado, esta deverá ser movida para a seção correspondente. Cada tarefa está dividida em campos no seguinte formato:

[  ][ ][PRIO] MOD: tarefa

- O primeiro campo ([  ]) define se a tarefa está em andamento ou terminada. No último caso, deve ser marcada com um Ok.
- O segundo campo ([ ]) deverá conter a letra identificadora do desenvolvedor que está resolvendo (ou já resolveu) a tarefa, sendo D = Daniel, G = Giovani.
- O terceiro campo ([PRIO]) indica a prioridade na resolução da tarefa. A prioridade 1 é a mais significativa na resolução. A 2 é a segunda mais significativa e a terceira é a menos significativa.
- O quarto campo (MOD: ) indica o módulo ao qual a tarefa pertence, sendo BAS = Básico, ACC = Contábil, SPR = Processo seletivo, INS = Institucional, FIN = Financeiro, OUT = Outros.
- O quinto campo (tarefa) indica a tarefa a ser realizada.
