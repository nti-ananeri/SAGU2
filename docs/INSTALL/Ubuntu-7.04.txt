CONFIGURAÇÕES:
==============

Locale
------
 É necessário adicionar a codificação "ISO-8859-1" e gerar novamente os locales do sistema. Para isso faça:
 # echo "pt_BR.ISO-8859-1 ISO-8859-1" >> /var/lib/locales/supported.d/pt
 # locale-gen
 # localedef -i pt_BR -c -f ISO-8859-1 pt_BR

Apache 2
--------
 # vim /etc/hosts
   > 127.0.0.1 sagu2local

 # vim /etc/apache2/sites-enabled/001-sagu2
   > <VirtualHost sagu2local:80>
   >   ServerAdmin admin@server.org
   >   DocumentRoot /usr/local/sagu2_RC5/html
   >   ServerName sagu2local
   > </VirtualHost>
   > <Directory "/usr/local/sagu2_RC5/html">
   >     AllowOverride All
   >     Options FollowSymLinks
   >     Order allow,deny
   >     Allow from all
   > </Directory>

 Editar o arquivo /etc/apache2/apache2.conf e descomentar a seguinte linha
   > AddDefaultCharset ISO-8859-1

PostgreSQL 8
------------
 Edite o arquivo /etc/postgresql/8.2/main/pg_hba.conf e altere a seguinte linha:
  > local   all         postgres                          ident sameuser
 por essa:
  > local   all         postgres                          trust

 Reinicie o postgres:
 # /etc/init.d/postgresql-8.2 restart

 # psql -Upostgres
   $ alter user postgres with password 'postgres';
   $ \q
 
 # vim /etc/postgresql/8.2/main/postgresql.conf
   > enable_seqscan = false
   > add_missing_from = on

PHP 5.2.0
---------
 # vim /etc/php5/apache2/php.ini
  > memory_limit = 128M
 # se, durante o uso do sagu, for exibida uma tela solicitando o salvamento do index.php, alterar a variável memory_limit, aumentando a memória disponível para o php.
 # terá uma variável chamada "error_reporting", apagar toda a referencia dela e colocar "E_ALL & ~E_NOTICE & ~E_WARNING"

Sagu2 RC5
---------
 # cd /usr/local/sagu2_RC5/sql
 # sh install.sh
 
 Caso queira tenha criado um VirtualHost diferente de "sagu2local" ou tenha instalado em um diretório diferente de "/usr/local/sagu2_RC5", configure o arquivo /usr/local/sagu2_RC5/etc/miolo.conf
 Caso a configuração do host do postgres seja diferente de "127.0.0.1", e/ou o usuário e a senha sejam diferentes de "postgres:postgres", altere os arquivos module.conf que estão em /usr/local/sagu2_RC5/modules/NOME_MODULO/etc

Permissões
----------
É necessário dar permissão para o usuário do Apache para duas pastas de upload utilizadas pelo Sagu2 e nos diretórios de log e base de dados do sqlite:
 # chown -R www-data /usr/local/sagu2_RC5/modules/basic/upload
 # chown -R www-data /usr/local/sagu2_RC5/modules/basic/html/images/upload
 # chown -R www-data /usr/local/sagu2_RC5/modules/admin/sql
 # chown -R www-data /usr/local/sagu2_RC5/var/log

FIREFOX
-------
Na URL digitar about:config
Procurar por dom.max_script_run_time e definit para 20 ou mais

Xvfb e OpenOffice.org
---------------------
   # cd /usr/local/sagu2_RC5
   # tar -zxvf tools/openoffice-config.org2.tar.gz
   # chown -R www-data .openoffice.org2

  Ainda deveremos editar o arquivo de configuração de impressoras do openOffice.org2 para adicionar as linhas abaixo no final do arquivo:
   # vim /usr/lib/openoffice/share/psprint/psprint.conf
   > ;geracao de pdf(agata+openoffice)
   > [Conversor de PDF]
   > Printer=SGENPRT/PDF
   > DefaultPrinter=0
   > Location=
   > Comment=
   > Command=/usr/bin/gs -q -dNOPAUSE -sDEVICE=pdfwrite -sOutputFile="(OUTFILE)" -
   > Features=pdf=/tmp/
   > Copies=1
   > Scale=100
   > Orientation=Portrait
   > PSLevel=0
   > ColorDevice=0
   > ColorDepth=24
   > MarginAdjust=0,0,0,0
   > PerformFontSubstitution=true
   > SubstFont_Arial=Helvetica
   > SubstFont_Times New Roman=Times
   > SubstFont_Cumberland=Courier
   > SubstFont_Timmons=Times
   > SubstFont_Helmet=Helvetica
   > SubstFont_Thorndale=Times
   > SubstFont_Albany=Helvetica
   > SubstFont_Courier New=Courier

  Depois disso, deveremos configurar o Xvfb. Basta criar o arquivo /etc/X10.host com o conteúdo "localhost"
   # echo 'localhost' > /etc/X10.host

AGATA
-----
 Editar o arquivo /usr/local/sagu2_RC5/agata/agata.ini e definir o parâmetro abaixo:
   > OpenofficeToPdf = export HOME=/usr/local/sagu2_RC5; /usr/bin/ooffice -display :10 -pt PDF

Xvfb:
 Adicionar a seguinte linha no arquivo /etc/rc.local, antes da linha "exit 0"
   > Xvfb :10 -auth /tmp/X10.host &

SUMÁRIO:
========
 O '#' é o caractere que indica o prompt de comandos do linux.
 O '$' é o carcatere que indica o prompt de comandos do psql, do PostgreSQL.
 O '>' é o caractere que indica que a linha deverá ser inserida ou modificada, caso já exista, no arquivo.
 Obs: Utilizar o usuário root em todo processo de instalação, configuração e inicialização dos programas.
